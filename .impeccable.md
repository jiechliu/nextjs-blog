## Design System

### File Structure

```
src/
├── styles/
│   ├── tokens.css     ← Design tokens (colors, spacing, typography, shadows)
│   │                    All values use OKLCH. Light + dark themes via [data-theme].
│   └── base.css       ← Base utility classes (.card, .btn-primary, .btn-secondary,
│                         .tag, .nav-link, .container-custom, .section-heading, etc.)
│
└── components/
    └── ui/
        ├── index.ts          ← Barrel export for all UI primitives
        ├── Tag.tsx           ← Tag pill (renders as link or span)
        ├── Button.tsx        ← Polymorphic button (button | Link | <a>)
        └── ThemeProvider.tsx ← Theme context, useTheme hook, ThemeToggle button
```

### How to activate tokens

1. Import `tokens.css` and `base.css` in `globals.css` (before Tailwind layers):
   ```css
   @import '../styles/tokens.css';
   @import '../styles/base.css';
   ```

2. Wrap the app in `<ThemeProvider>` (inside `layout.tsx`, replace or augment
   `ClientWrapper`). Add `<ThemeToggle />` to `Header.tsx`.

3. Replace hardcoded colors/gradients with CSS variables:
   - `#374151` → `var(--color-text)`
   - `rgba(255,255,255,0.7)` backdrop → `var(--color-bg-elevated)` + `border: 1px solid var(--color-border)`
   - `.title-gradient` (banned gradient text) → plain `color: var(--color-text)` with font-weight variation

### Banned patterns (already in globals.css — to remove)

- `.title-gradient` — gradient text via background-clip (banned)
- `body::before` radial-gradient decoration blobs
- `.card` glassmorphism (backdrop-filter + rgba bg) — replaced by token-based card
- `.neon-text`, `.rainbow-border`, `.glitch-effect`, `.dynamic-gradient`
- `float-animation`, `pulse-glow`, `wave-animation`, `particle-bg` — purely decorative
- `magnetic-hover` scale(1.02) — too aggressive, remove

## Design Context

### Users

三类核心读者：
1. **同行开发者**：下班后深夜在家阅读，低光环境，需要深色模式友好，眼睛疲劳容忍度低，讨厌分散注意力的动效
2. **开发者新人**：白天查阅教程，需要清晰的信息层级和易扫描的排版，代码块可读性优先
3. **作者本人**：知识记录/笔记，需要快速定位内容，目录导航和标签分类至关重要

核心 Job：快速找到并沉浸阅读技术文章，代码示例要清晰易复制，信息密度高但不压迫。

### Brand Personality

**品牌名**：BlockCoder（方块世界的代码师）——有游戏感（Minecraft 内核）但落地是工程师写作
**三词定位**：冷静、精确、专业
**情感目标**：读者打开页面时感到「这里的内容值得信任」，阅读中感到「安静、专注、沉浸」，离开时感到「有所得」

### Aesthetic Direction

**视觉基调**：编辑型（Editorial）+ 工程精密（Engineering Precision）。参考：
- 好的方向：Stripe Docs 的信息密度感、Linear 的深色精致感、Ink & Switch 的文字密度与余白并存
- 反面参考（明确避免）：过度 glassmorphism、紫粉渐变、霓虹灯文字、彩虹边框、大量 CSS 动画特效——这些正是当前博客需要克制的东西

**主题**：支持深浅切换。
- 深色模式：主色调 near-black（带极轻暖色调，不是纯黑），适合深夜阅读
- 浅色模式：冷白底（带极轻蓝灰色调），干净、呼吸感强
- 两种模式下强调色保持一致：蓝灰或冷靛色（远离紫粉）

**字体方向**：
- 中文：思源宋体（Noto Serif SC）或苦禅定宋——衬线字体给正文带来阅读节奏，区别于所有「无衬线 Sans 技术博客」
- 英文/代码展示：Berkeley Mono 或 Commit Mono——非 IBM Plex Mono，避免泛滥
- 标题：考虑 Geist 或 Cabinet Grotesk（非 Inter/DM Sans）

**颜色系统**（OKLCH）：
- 背景（暗）：oklch(12% 0.008 250)——近黑带极轻冷色调
- 背景（亮）：oklch(97% 0.005 240)——近白带极轻蓝灰
- 强调色：oklch(62% 0.12 255)——冷靛蓝，不是紫色
- 代码背景（暗中）：oklch(18% 0.010 250)
- 正文色（暗）：oklch(85% 0.008 240)

### Design Principles

1. **内容第一，装饰消失**：任何视觉元素存在的理由只有一个——让文字更易读、层级更清晰。动效仅用于状态变化，不做装饰。
2. **代码块是一等公民**：代码高亮、字体、复制按钮、行号都需要精心设计，这是技术博客的核心产品。
3. **余白即设计**：用充足的 spacing 建立信息层级，不靠卡片、边框、阴影堆叠。
4. **深浅模式平等对待**：不把深色模式当「可选附加」，两种模式都是一等公民，分别设计而不是反色。
5. **克制的个性**：品牌感体现在排版细节和一致性上，而非装饰特效——BlockCoder 这个名字本身足够有个性。
